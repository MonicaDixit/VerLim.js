Demo : #1 Adding VerLim.js to see the height of page in form of progress bar

follow the below steps
  • Post adding reference to the VerLim.js and JQuery simply add the below code snippet

    • $().VerLim();
Very easy isn't it..??? Now if you observe this page while scrolling; you will find this demo working.(internet connection required in order to load JQuery.js)
There are lots of customisable options available to change progress bar and its functionality; Navigate here for the complete list of customisation options.

Demo : #2 Using themes to customise progress bar

In order to do so follow the below steps
  • Add reference to JQuery and VerLim.js
  • Get any theme of your choice (available in github) and put reference in the code
  • Pass configuration to VerLim as set theme : "on"

    • $().VerLim({theme : "on"});
Done!!! that's it, VerLim will take care of rest.
Want to make your own theme? you are free to to so modify the existing one or in css make .VerLim class and put your imagination there.

Demo : #3 Scrolling to a desired position using scrollTo method

Imagine you are visiting a page in mobile device & don't know how lengthy the page is??? well!! VerLim.js solves this by showing progress bar, but; what if you are reading the same page and want to navigate to the position where you left in the last read?
scrollTo method works very well in that cases, lets see how..
  • Add reference to JQuery and VerLim.js
  • In the below example we are calling the scrollTo method on onClick event of the button to go to specific position, you can use it according to your requirement
  • to call scrollTo, use this code snippet

    • $('body').scrollTo(50);
  • We have passed the body as selector because we want to navigate to the body's specific position, (you can use it to navigate to any scrollable element, in a later demo)
  • You can pass any number between 1 to 100; scrollTo will treat it as percentage (as shown by the plug-in on the right top), and proceed further.
Awesome!!! Isn't it?

Demo : #4 scrollTo method with scroll duration

scrollTo is not limited to this, you can even set the animation time by passing duration amount like this:

$('body').scrollTo(50,{duration: 5000});

lets see how its looks running

Demo : #5 scrollTo method with callback

Pretty cool huh!!!, not only this you can call a call back method after the animation completes, to do so pass your call back function in configuration like this:

$('body').scrollTo(50,{duration:5000},function(){alert('Hello World!!!')});

or

 $('body').scrollTo(50,function(){alert('Hello World!!!')});

Demo : #6 scrollTo method with easing options

OK.. That's good, but what if someone wants to change easing style? Currently the core JQuery library includes "linear" and "swing". To use other easing methods you will need to extend your easing methods via separate plug-in, In this demo we are using easing.js which have lots of easing option (my personal favourite)

 $('body').scrollTo(50,{duration:5000, easing:"easeOutBounce"}});

or

 $('body').scrollTo(50,{easing:"easeInOutBack"}});


Demo : #7 usingscrollTo with other scrollable elements

Well, VerLim's scrollTois not limited to just scroll the window, you can use the same to scroll any scrollable element (e.g. div, section etc.). To do so you have to pass the scrollable element in the selection where we used to pass 'body' ;with selection prefix (e.g. # for id . for class), rest is same.

 $('#yourSelection').scrollTo(50,{duration:1000, easing:"easeOutBounce"}});

Demo : #9 How much amount of page i have scrolled, to put some conditon on a specific amount

VerLim shows the progress bar but what if someone wants to use the percentage in her code? e.g. after 70% scrolling one want to show some ads or banner,One can use the below code to get the scroll percentage

 $('#scroller').attr('scroller-width');

This will return a number on which condition can be applied, (when you scroll this page to 80% a small banner on the right bottom is the example of this)
Now adding a scroll to top button in a website can also be achieved by this by writing very less amount of code.
